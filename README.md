# Network Monitoring Dashboard

Веб-приложение для мониторинга сетевого соединения в реальном времени.

## Возможности

- ✅ Проверка доступности интернета
- ✅ Измерение скорости загрузки/отдачи
- ✅ Мониторинг пинга
- ✅ Сбор статистики по использованию трафика
- ✅ Логирование событий отключения
- ✅ Отображение внешнего IP-адреса
- ✅ Отображение локального IP- и MAC-адреса
- ✅ Информация о сетевом интерфейсе
- ✅ Отображение имени хоста
- ✅ Определение провайдера
- ✅ Список процессов, использующих сеть

## Требования

- Python 3.8+
- Docker (опционально)

## Установка и запуск

### Способ 1: Локальная установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/avkolchin1995/internet-monitor-dashboard.git
cd internet-monitor-dashboard
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Запустите приложение:
```bash
python app.py
```

4. Откройте в браузере: `http://localhost:5000`

### Способ 2: Запуск через Docker

1. Соберите Docker образ:
```bash
docker build -t network-monitor .
```

2. Запустите контейнер:
```bash
docker run -d -p 5000:5000 --name network-monitor network-monitor
```

3. Откройте в браузере: `http://localhost:5000`

## Использование

- Приложение автоматически обновляет данные каждые 10 секунд
- Все события логируются в файл `internet_events.log`
- Интерфейс адаптирован для мобильных устройств
- Скорость интернета измеряется при наличии соединения

### Ключевые особенности реализации:

1. **Асинхронность**: Speedtest выполняется в отдельном потоке, чтобы не блокировать основной поток мониторинга.

2. **Кеширование**: Данные кешируются на 5 секунд для уменьшения нагрузки на систему.

3. **Обработка ошибок**: Все исключения перехватываются и логируются.

4. **Логирование**: Подробные логи всех событий с таймстемпами.

5. **Автообновление**: Frontend автоматически обновляется каждые 10 секунд через JavaScript.

6. **Адаптивный дизайн**: Интерфейс работает на всех устройствах.

7. **Безопасность**: В Docker контейнере приложение запускается от непривилегированного пользователя.

8. **Производительность**: Используются легковесные библиотеки и оптимизированные запросы.

Этот проект готов к использованию и может быть развернут как локально, так и в Docker контейнере. Он предоставляет полный мониторинг сетевого соединения в реальном времени с красивым и функциональным интерфейсом.
